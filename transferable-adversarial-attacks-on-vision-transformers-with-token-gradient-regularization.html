Transferable Adversarial Attacks on Vision Transformers with Token Gradient Regularization

Szl

CVPR 2023

Introduction

本文认为在梯度反向传播的过程中，如果梯度方差大那么有害最终生成的对抗样本的迁移性。故提出一种梯度正则化的方法来提升对抗样本的迁移性。具体的本文贡献如下：

  • 提出了一种基于令牌梯度正则化（TGR）的可转移对抗性 ViTs 攻击方法。根据 ViTs 的体系结构，TGR 对 ViTs 内部各块中的反向传播梯度进行了标记化正则化，并利用正则化梯度生成对抗样本。
  • 我们进行了广泛的实验来验证我们方法的有效性。实验结果证实，平均而言，我们的方法可以超越最先进的攻击方法，在攻击 ViT 模型上有 8.8% 的明显优势，在攻击CNN模型上有 6.2% 的优势。
  • 我们表明，我们的方法可以与其他兼容的攻击算法相结合，以进一步提高生成的对抗性样本的可转移性。我们的方法还可以扩展到使用CNN作为本地源模型。

Method

我们假定初始图像 $x$ 的大小为 $H\times W\times C$。ViTs 将图像分割为多个补丁 $x_p={x_p^1,x_p^2,\cdots,x_p^n}$，其中 $x_P^i$ 代表原图的第 $i$ 个补丁。每个补丁 $x_P^1$ 的 shape 为 $P\times P\times C$，其中 $P$ 为补丁的 Size。一共有 $N=\frac{H\cdot W}{P^2}$ 个补丁。图像的真实标签为 $y$。我们将 DNN 分类定义为 $f(x)$。$J(x,y;f)$ 表示分类器 $f$ 的分类损失函数，例如交叉熵损失。给定一个目标图像 $x$ ，对抗攻击的目标是找到一个对抗样本 $x^{adv}$，这个对抗样本可以误导分类器，比如：在人类难以察觉的情况下实现 $f(x^{adv})\neq f(x)$。

Token Gradient Regularization

以往的梯度正则化只关注输入图像上的梯度，而忽略了模型结构内的梯度。本文关注正则化模型内部的梯度。

为了减小 ViTs 内部反向传播梯度的方差，我们寻求正则化梯度并考虑 ViTs 的结构特征。令牌是 ViTs 中基本构建块，我们把对应于 ViTs 内部块中的一个令牌的反向传播梯度称为令牌梯度。因此，对中间块的梯度进行正则化就是对令牌梯度进行正则化。具体来说，如果一个令牌的反向传播梯度在所有令牌中处于 top-k 或者 bottom-k 梯度量级，那么称它为极端令牌，其中 k 是一个超参数。


图1:说明我们的令牌梯度正则化(TGR)方法。红色条目表示具有极值的反向传播梯度。对应于vit内部块中的一个令牌的反向传播梯度称为令牌梯度。由于我们以标记方式正则化反向传播梯度，因此我们消除了反向传播期间极端梯度所在的标记梯度(用叉标记)，以减少梯度方差。然后我们使用正则化梯度生成对抗性样本。
Implementation

我们正则化 ViTs 结构中的三个组件：注意力块中注意力和 QKV 组件，MLP 块中的 MLP 组件。

Attention Component.

注意力组件使用多头自注意力机制来计算令牌之间的相关性。我们假设在注意力组件中有 M 个自注意力计算。因此，对于每一张图像将会产生 $H\times H\times M$ 的注意力图，其中 $N$ 代表补丁令牌的数量。我们认为自注意力头分别计算令牌间的相似度。因此，我们独立地对注意力分量的每个输出通道中的反向梯度进行排序。我们首先定位出注意力图中的极端令牌并标出它们的位置。然后，我们去除它们所在的行和所在的列的全部梯度。

QKV Component.

假设 QKV 组件具有 C 个通道，对于每张图像 QKV 组件的 size 为 $N\times C$。我们同样认为通道之间是独立的，我们独立得对 QKV 组件每个通道的反向梯度进行排序。我们定义具有 top-k 和 bottom-k 梯度得令牌是极端令牌。因此，我们剔除全部得极端令牌梯度。

MLP Block.

MLP 块沿着通道聚合每个令牌的信息。我们假设 MLP 块具有 C 通道，因此对于一张图像，MLP 块的大小为 $N\times C$。与 QKV 组件类似，我们对 MLP 组件的每个输入通道独立进行优先级排序。我们消除了具有top-k 或 bottom-k 反向传播梯度幅度的令牌的极端梯度条目。

TGR在每个组件上的说明如图1所示。除了正则化极端令牌之外，我们还在每个组件上引入缩放因子 s 以减少整体梯度方差。式1为步骤t的对抗性攻击。式中，$g’$ 为输入的正则化后向梯度。modules 表示模型结构，gradients 记录网络中的反向梯度。$TGR(·)$ 为Token Gradient Regularization方法，具体见算法1。 $$ \begin{aligned} & g’=TGR(Grads,modules,k,s) \ & x_{t+1}^{adv}=x_t^{adv}+\alpha\cdot sgn{g’} \end{aligned}\tag{1} $$ 其中 $\alpha$ 是一个超参数来控制每次迭代中更新的步长。
