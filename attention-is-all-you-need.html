Attention Is All You Need

Abstract

目前主流的序列推导模型是基于复杂的循环或者包含边解码器的卷积神经网络。效果最好的模型通过注意力机制连接编解码器。我们提出一种新型的简单网络结构—-Transformer，仅基于注意力机制，完全移除了循环和卷积结构。对两个机器翻译任务的实验表明，这些模型在质量上更优越，同时更可并行化，并且需要的训练时间显着减少。对模型质量和训练速度的提升进行详细介绍。

1 Introduction

介绍了循环神经网络，长短期记忆和门控循环神经网络。指出这些网络不适合并行化。

本文提出了 Transformer ，这是一种模型架构，避免了递归，而是完全依赖注意力机制来描绘输入和输出之间的全局依赖关系。Transformer 运行更多的并行化。

2 Background

指出目前的模型例如 ByteNet 和 ConvS2S ，关联模型中任意两个位置的输出所需要的操作数随着这两个位置的增长而增长，ConvS2S 是线性增长而 ByteNet 是指数级增长。这就导致了模型很难学习距离较远位置元素之间的关联。在 Transformer 只需要常数的操作数，尽管代价是由于注意力加权位置的平均化而降低了有效分辨率，我们在 3.2 节中将会描述如何使用多头注意力来抵消这种影响。

自注意力也称内部注意力是一种与单一序列的不同位置相关的注意机制，以便计算出序列的表征。介绍了一下自注意力机制的应用。

端到端记忆网络是基于递归注意机制，而不是顺序排列的递归，并已被证明在简单语言问题回答和语言建模任务中表现良好。

然而，据我们所知，Transformer 是第一个完全依靠自我注意来计算其输入和输出的表征而不使用序列对齐的RNN 或卷积的转导模型。在下面的章节中，我们将描述 Transformer，激励自我注意，并讨论与其他模型相比的优势。

3 Model Architecture

大多数复杂的神经序列推导模型具有编解码器结构。编码器把由符号表示$(x_{1},…,x_{n})$的输入序列映射到一系列的连续表示$z=(z_{1},…,z_{n})$ 。给定 $z$ ，解码器一个接一个生成输出输出序列符号 $(y_{1},…,y_{n})$ 。在每一步，模型都是自回归的，当生成下一个符号时会使用之前产生的符号作为附加输入。

Transformer 遵循这种整体架构，对编码器和解码器使用堆叠的自注意力和逐点全连接层，分别如图 1 的左右半部分所示。


图 1:Transformer 模型结构
3.1 Encoder and Decoder Stacks

Encoder ：编码器由 $N=6$ 个相同的层组成。每一层具有两个子层。第一层是多头注意力机制，第二层是简单的、位置上的全连接的前馈网络。我们在每个子层前后使用残差连接之后是层归一化。也就是说，每个子层的输出就是 $\text{LayerNorm}(x+\text{SubLayer}(x))$ ，其中 $\text{SubLayer}(x)$ 是子层本身所表示的函数。为了便于这些残差连接，模型中所有的子层都会产生维度 $d_{\text{model}}=512$ 的输出。

Decoder：解码器同样由 $N=6$ 个相同的层组成。除了每个编码器层中的两个子层之外，解码器还插入了第三个子层，他对编码器堆栈的输出执行多头注意力。和编码器一样，我们使用残差连接每一个子层的前后，之后是层归一化。我们还修改了解码器堆栈中的自注意力子层，以防止位置关注后续位置。这种掩码与输出嵌入位置偏移的组合确保对位置 i 的预测只取决于小于 i 的位置的已知输出。

3.2 Attention

注意力的功能可以被描述为把 query 和一组 key-value 对映射成一个输出，其中 query、key、value 和输出都是向量。输出是由 value 加权求和得到，其中分配给每个 value 的权重是由 query 和相应的 key 的兼容函数计算出来的。


图 2:(左)缩放点积注意力。（右）多头注意力由并行运行的几个注意力层组成。
3.2.1 Scaled Dot-Product Attention

我们称我们这种特殊的注意力机制为“缩放点积注意力（Scaled Dot-Product Attention）”（如图二）。输入由 query、维度为 $d_{k}$ 的 keys 和维度为 $d_{v}$ 的 values 组成。我们在 query 和所有的 keys 之间做点乘，之后除以 $\sqrt{d_{k}}$ 然后经过 softmax 函数后获得的值做为 values 的权重。

在实际操作中，我们计算注意力函数同时在一组 query 上，将它们打包成一个矩阵 $Q$ 。 keys 和 values 同样打包成矩阵 $K$ 和 $V$ 。我们使用下式计算结果矩阵： $$ \text{Attention}(Q, K, V)=\text{softmax}(\frac{QK^{T}}{\sqrt{d_{k}}})V $$ 两种最常用的注意力函数是加法注意力和点积（乘法）注意力。除了 $\frac{1}{\sqrt{d_{k}}}$ 外点积注意力与我们的算法相同。加法注意力包含一个隐藏层的前馈函数计算兼容性函数。尽管两种方式具有相似的理论复杂度，但是点乘注意力使用编码上更加优化的矩阵乘法所以在实践中运行更快并有更高的空间利用率。

虽然对于较小的 $d_{k}$ 两者的表现相似，但在较大的 $d_{k}$ 时，加法注意力要优于没有缩放机制的点乘注意力。我们认为在较大的 $d_{k}$ 时，点乘以数量级增长，将 softmax 函数推入梯度极小的区域。为了抵抗这种影响，我们使用 $\frac{1}{\sqrt{d_{k}}}$ 缩放点乘结果。

3.2.2 Multi-Head Attention

我们发现与其用 $d_{\text{model}}$ 维的 keys、values 和 queries 来执行一个单一的注意力函数，不如用不同的、学习过的线性投影来将 queries、keys 和 values 分别投影到 $d_{k}$、$d_{k}$ 和 $d_{v}$ 维。在每个 queries、keys 和 values 的每个投影版本上，我们并行执行注意力函数，产生 $d_{v}$ 维输出值。这些数据被连接起来并再次进行投影，从而得到最终值，如图二所示

多头注意允许模型在不同的位置上共同关注来自不同表征子空间的信息。在单头注意的情况下，平均化抑制了这一点。

其中投影由参数矩阵 $W_{i}^{Q}\in\mathbb{R}^{d_{\text{model}}\times d_{k}}$ ，$W_{i}^{K}\in\mathbb{R}^{d_{\text{model}}\times d_{k}}$，$W_{i}^{V}\in\mathbb{R}^{d_{\text{model}}\times d_{v}}$ 和 $W^{O}\in\mathbb{R}^{hd_{v}\times d_{\text{model}}}$ 确定。

在本文中我们包含 $h=8$ 的并行注意力层或者说头。对于它们的每一个我们使用 $d_{k}=d_{v}=d_{\text{model}}/h=64$ 。由于在每一个头中都减小了维度，总的计算消耗小于在单个头上计算完整的维度。

3.2.3 Applications of Attention in our Model

Transformer 使用多头注意力的三种方式：

  • 在“编码-解码注意力”层，queries 由上一解码层得到并且存储由编码器输出的 keys 和 values。这使得解码器中的每个位置都能在输入序列的所有位置上参加。这模拟了序列到序列模型中典型的编码器-解码器注意机制。
  • 编码器包含自注意力层。在自注意力层中所有的 keys、values 和 queries 都来之相同的地方，在本例中，所有的输出来自前一层的编码器。编码器中的每个位置都能关注到编码器上一层的所有位置。
  • 同样的，解码器中的自注意力层允许所有位置都能关注到包含自身在内的所有位置。我们需要防止信息流在在解码器中左移动以保持自回归特性。我们通过在缩放点积注意力中屏蔽（设置为$-\infty$） softmax 输入中对应与非法连接的所有值。如图二所示。
3.3 Position-wise Feed-Forward Networks

除了注意力子层，在编解码器的所有层包含一个全连接的前馈神经网络，它对于每个位置都是独立且相同的。这个全连接网络包含两个线性转换并在中间使用 ReLU 激活函数。 $$ \text{FFN}(x)=\max(0, xW_{1}+b_{1})W_{2}+b_{2} $$ 在不同的位置上使用相同的类型的线性变换，但是在层与层之间是不同的。另外一种描述是使用两个核大小为1的卷积。其中输入输出的维度均为 $d_{\text{model}}=512$ ，中间层的维度为 $d_{ff}=2048$ 。

3.4 Embeddings and Softmax

与其他序列转换模型类似，我们使用被学习过的嵌入把输入和输出标记（tokens）转换成维度为 $d_{\text{model}}$ 的向量。我们还使用常用的线性变换和 softmax 函数把编码器的输出转换成被预测的下一个标记（tokens）的可能性。在我们的模型中，我们在两个嵌入层和 pre-softmax 线性变换中共享权重矩阵，类似此论文。在嵌入层我们把这些权重乘上 $\sqrt{d_{\text{model}}}$。

3.5 Positional Encoding

由于我们的模型不包含递归和卷积，为了让模型利用序列的顺序，必须注入一些关于序列中标记（tokens）的相对或绝对位置信息。为此，我们在编解码器栈底部的输入嵌入中添加了 “Positional Encoding（位置编码）”。位置编码与嵌入（embeddings）具有相同的维度 $d_{\text{model}}$ ，所以这两者可以进行相加。实现位置编码有许多选择，可学习的或固定的。

在本文中，我们使用不同频率的正弦和余弦函数： $$ PE_{(pos, 2i)}=sin(pos/10000^{2i/d_{model}}) $$ $$ PE_{(pos,2i+1)}=cos(pos/10000^{2i/d_{model}}) $$ 其中 $pos$ 是位置，$i$ 是维度。也就是说，位置编码的每个维度对应一个正弦信号。波长从 $2\pi$ 到 $10000\cdot 2\pi$ 的几何级数。我们假设这个函数能使模型能够容易学习去关注相对位置关系，因为对于任何固定的偏置量 $k$，$PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性函数。

我们还尝试使用了可学习的位置嵌入，最终两者产生了几乎相同的结果（见表3行E）。我们选择了正弦版本，因为它能够处理比训练时更长的序列。

4 Why Self-Attention

在本节中我们比较了自注意力层与递归和卷积层进行比较，这些层通常用来把可变长度的符号表示序列 $(x_{1},…,x_{n})$ 转换成另一个长度相等的序列 $(z_{1},…,z_{n})$，其中 $x_{i},z_{i}\in\mathbb{R}^{d}$ ，比如典型序列转导编码或解码器中的隐藏层。使用自注意力是由于三个需求。

第一是每一层的总计算复杂度。第二是计算的并行化程度，以所需的最小顺序操作数衡量。

第三是网络中远程依赖关系之间的路径长度。学习远程依赖是许多序列转导任务中的一项关键挑战。影响学习这种依赖关系能力的一个关键因素是向前和向后信号在网络中必须遍历的路径长度。输入和输出序列中任意位置组合之间的路径越短，就越容易学习远程依赖关系。因此我们比较了不同神经网络层中任意两个输入和输出的最大路径长度。

如表 1 所示，自注意力层连接所有的位置使用常数级的顺序操作，然而循环层需要 $O(n)$ 的顺序操作。在操作复杂度方面，当序列长度 $n$ 小于表示维度 $d$ 时（这是在机器翻译中，最先进的模型使用的句子表示例如 word-piece 和 byte-pair 表示通常出现的情况）注意力层比循环层更快。为了提高涉及很长序列的任务的计算性能，自注意力可以被限制为只考虑输入序列中以各自输出位置为中心的大小为r的邻域。这将会增加最大路径长度到 $O(n/r)$ 。我们计划在未来的工作中进一步研究这种方法。

单一的卷积层当 $k<n$ 时无法连接所有位置上的输入输出对。为了连接任意位置在连续核的情况下需要有 $O(n/k)$ 个卷积层堆栈或者需要 $O(log_{k}(n))$ 层扩张卷积（dilated convolutions），同时增加网络中任意两个位置之间的最长路径的长度。卷积层通常比循环层更加昂贵，相差 $k$ 倍。虽然，可分离卷积大幅减少了复杂度至 $O(k\cdot n\cdot d+n\cdot d^{2})$ 。但是当 $k=n$ 时，可分离卷积的复杂度与一个自注意层加一个逐点前馈层（point-wise feed-forward layer）相等，这是在本模型中使用的操作。

作为附带的好处，自注意力层可以产生更具有解释性的模型。我们检查模型和注意力分布，并在附录中展示和讨论示例。单个注意力头明显学会了执行不同的任务，而且多头注意力似乎还表现出与句子的句法和语义结构有关的行为。
